Digital Driver Dashboard

Role: Developer — C++, Embedded Software • Tech: C++, CAN, Raspberry Pi, Multithreading

Developed a high-frequency (60Hz) real-time driver dashboard that decodes CAN bus signals, structures telemetry for concurrent access and displays key metrics for driver feedback and rapid diagnostics.

  • Implemented a multithreaded C++ consumer for CAN frames with lock-minimised concurrent data structures to feed the UI at 60Hz.
  • Built an on-vehicle UI optimized for readability under motion and low-latency updates.
  • Created offline test harnesses to simulate CAN traffic for robust validation away from the vehicle.

Impact: Enabled live driver feedback and rapid fault diagnosis during testing sessions; shortened debugging cycles.

CAN Testbench

Role: Developer — Python, Test Automation • Tech: Python, SocketCAN, pytest

Built a Python-based testbench to simulate CAN traffic and validate telemetry consumer code offline, enabling faster development and safer on-car testing.

  • Designed scenario-based CAN traffic generators and unit-tests to exercise dashboard and data consumers.
  • Implemented logging/analysis tools to replay test sessions and trace data issues.
  • Used pytest-driven tests to ensure regressions were caught before on-vehicle validation.

Impact: Accelerated development cycles and reduced integration issues during track testing.

HV Safety & Validation

Role: Lead — Electrical validation, process design

Led HV validation and developed safety procedures for battery and HV distribution on the electric vehicle. Documentation and processes were commended by FSUK scrutineers.

  • Authored validation procedures for battery assembly, insulation checks and safety interlocks.
  • Trained electrical safety officers and established pre-test checklists.
  • Coordinated testing sessions with data capture for verification.

Impact: Enabled safe test sessions and produced documentation for team continuity.
